# Исправление мобильной версии и Best Practices для изображений

## Проблема: Inspect показывает не то же самое, что на реальном устройстве

### Причины:
1. **Viewport meta тег** - должен быть правильно настроен в теме WordPress
2. **Медиа-запросы** - должны использовать `device-width` вместо фиксированных пикселей
3. **Размеры экрана** - реальные устройства имеют разные pixel density (retina, etc.)

### Решение:

#### 1. Проверьте viewport meta тег в теме
Должен быть в `header.php` или через `wp_head()`:
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=5.0, user-scalable=yes">
```

#### 2. Используйте правильные медиа-запросы
В CSS уже используются правильные breakpoints:
- `@media (max-width: 560px)` - мобильные телефоны
- `@media (max-width: 900px)` - планшеты
- `@media (max-width: 1200px)` - маленькие десктопы

#### 3. Для тестирования в Chrome DevTools:
- Используйте **Device Toolbar** (Ctrl+Shift+M / Cmd+Shift+M)
- Выберите конкретное устройство (iPhone, Samsung, etc.)
- Включите **"Show device frame"** для более точного отображения
- Проверьте **"Throttling"** - установите "Slow 3G" для реалистичного тестирования

#### 4. Проверка на реальном устройстве:
- Используйте **Chrome Remote Debugging** для Android
- Используйте **Safari Web Inspector** для iOS
- Или просто откройте сайт на реальном устройстве

## Best Practices для изображений блюд

### 1. Рекомендуемые размеры изображений:

**Для карточек меню:**
- **Оптимальный размер:** 800x600px (соотношение 4:3)
- **Минимальный размер:** 400x300px
- **Максимальный размер:** 1200x900px (для retina дисплеев)
- **Формат:** WebP (лучше) или JPEG (совместимость)
- **Качество:** 80-85% (баланс между размером и качеством)

**Для модального окна (полноэкранный просмотр):**
- **Оптимальный размер:** 1920x1440px
- **Максимальный размер файла:** 500KB (для быстрой загрузки)

### 2. Оптимизация изображений:

```bash
# Используйте инструменты для оптимизации:
# - ImageOptim (Mac)
# - TinyPNG (онлайн)
# - Squoosh (Google, онлайн)
# - WordPress плагин: Smush или ShortPixel
```

### 3. Alt текст для SEO:

**Хорошие примеры:**
- ✅ "Блинчики с домашним вареньем на белой тарелке"
- ✅ "Драники с сметаной и зеленью, вид сверху"
- ✅ "Котлеты из говядины с гарниром из овощей"

**Плохие примеры:**
- ❌ "Фото 1"
- ❌ "Изображение"
- ❌ "IMG_1234"

### 4. Загрузка изображений:

**Вариант 1: WordPress Media Library (рекомендуется)**
1. Загрузите фото в Media Library
2. Скопируйте URL изображения
3. Вставьте в поле "Фото (URL)"
4. WordPress автоматически создаст разные размеры

**Вариант 2: Внешний хостинг**
- Используйте CDN (Cloudflare, Cloudinary)
- Убедитесь что изображения оптимизированы
- Проверьте CORS настройки если нужно

### 5. Lazy Loading:

Уже реализовано через `loading="lazy"` в коде. Это означает:
- Изображения загружаются только когда пользователь прокручивает к ним
- Улучшает скорость загрузки страницы
- Экономит трафик на мобильных устройствах

### 6. Responsive Images:

Текущая реализация использует:
- `object-fit: cover` - изображение заполняет контейнер без искажений
- Фиксированная высота: 200px (десктоп), 180px (мобилка)
- Автоматическое масштабирование через CSS

### 7. Модальное окно для просмотра:

**Преимущества:**
- ✅ Пользователь может увидеть фото в полном размере
- ✅ Не нужно обрезать фото для карточек
- ✅ Лучший UX - можно кликнуть для детального просмотра

**Реализовано:**
- Клик на фото открывает модальное окно
- Закрытие по клику на фон или кнопку X
- Закрытие по Escape
- Адаптивный размер для мобильных устройств

## Рекомендации по workflow:

1. **Подготовка фото:**
   - Снимайте/готовьте фото в высоком разрешении
   - Обрезайте до соотношения 4:3
   - Оптимизируйте через ImageOptim или TinyPNG

2. **Загрузка:**
   - Загружайте в WordPress Media Library
   - Используйте описательные названия файлов: `блинчики-с-вареньем.jpg`

3. **Заполнение полей:**
   - URL: копируйте из Media Library
   - Alt текст: опишите что на фото (для SEO и доступности)
   - КБЖУ: используйте единый формат

4. **Тестирование:**
   - Проверьте на реальном мобильном устройстве
   - Проверьте скорость загрузки (PageSpeed Insights)
   - Проверьте отображение на разных размерах экрана
