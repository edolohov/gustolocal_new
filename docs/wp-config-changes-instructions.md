# Инструкция: Как добавить логирование в wp-config.php

## Проблема
В вашем `debug.log` старые данные за 2024 год, потому что `WP_DEBUG` был выключен (`false`). Новые ошибки не логировались.

## Решение

### Шаг 1: Найдите эти строки в вашем wp-config.php

```php
define('WP_HOME', 'https://gustolocal.es');
define('WP_SITEURL', 'https://gustolocal.es');

define('WP_DEBUG', false);  // ← ЭТУ СТРОКУ НУЖНО ИЗМЕНИТЬ

define( 'WP_MEMORY_LIMIT', '256M' );
define( 'WP_MAX_MEMORY_LIMIT', '512M' );

/* That's all, stop editing! Happy publishing. */
```

### Шаг 2: Замените на эти строки

**Замените:**
```php
define('WP_DEBUG', false);
```

**На:**
```php
// ============================================
// НАСТРОЙКИ ОТЛАДКИ И ЛОГИРОВАНИЯ
// ============================================
define('WP_DEBUG', true);              // Включить режим отладки
define('WP_DEBUG_LOG', true);          // Логировать ошибки в файл wp-content/debug.log
define('WP_DEBUG_DISPLAY', false);     // НЕ показывать ошибки на сайте (только в логах)
@ini_set('display_errors', 0);         // Отключить вывод ошибок на экран
```

### Шаг 3: Итоговый результат должен выглядеть так

```php
define('WP_HOME', 'https://gustolocal.es');
define('WP_SITEURL', 'https://gustolocal.es');

// ============================================
// НАСТРОЙКИ ОТЛАДКИ И ЛОГИРОВАНИЯ
// ============================================
define('WP_DEBUG', true);              // Включить режим отладки
define('WP_DEBUG_LOG', true);          // Логировать ошибки в файл wp-content/debug.log
define('WP_DEBUG_DISPLAY', false);     // НЕ показывать ошибки на сайте (только в логах)
@ini_set('display_errors', 0);         // Отключить вывод ошибок на экран

define( 'WP_MEMORY_LIMIT', '256M' );
define( 'WP_MAX_MEMORY_LIMIT', '512M' );

/* That's all, stop editing! Happy publishing. */
```

## Важно!

1. **`WP_DEBUG_DISPLAY` должен быть `false`** - чтобы ошибки не показывались посетителям сайта
2. **Ошибки будут записываться только в файл** `/wp-content/debug.log`
3. **После диагностики отключите отладку** - верните `WP_DEBUG` в `false`

## Где найти логи после включения

После включения логирования новые ошибки будут записываться в:
- **`/wp-content/debug.log`** - основной лог WordPress

Старые данные в файле можно:
- Удалить (очистить файл)
- Или оставить - новые ошибки будут добавляться в конец файла

## Проверка работы

1. Сохраните изменения в `wp-config.php`
2. Откройте сайт (если он зависает - это нормально, ошибки будут в логах)
3. Проверьте файл `/wp-content/debug.log` - должны появиться новые записи с текущей датой

## После решения проблемы

Когда найдете и исправите проблему, **отключите отладку**:

```php
define('WP_DEBUG', false);
define('WP_DEBUG_LOG', false);
```

Или просто удалите/закомментируйте добавленные строки.
